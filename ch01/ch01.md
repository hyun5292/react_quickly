# React 살펴보기

```
# 1-1 React란 무엇인가?
  > UI 컴포넌트 라이브러리
	  > React UI 컴포넌트
	    >> 매우 독립적이며 특정 관심사에 집중된 기능 블록
	    >> 자바스크립트를 이용
```

```
# 1-2 React가 해결할 수 있는 문제
  > 복잡한 웹 UI로 구성된 프론트엔드 애플리케이션을 개발하고 관리하는 데 어려움을 겪고 있음
  > 시간에 따라 변화하는 데이터를 다루는 거대한 어플리케이션의 개발
```

```
# 1-3 React의 장점
	## 1-3-1 간결성
		> 선언형 스타일 채택
			>> 선언형 스타일
				: 실행 결과가 어떻게 되어야 할지를 작성
				(명령형 스타일; 개발자가 순서대로 무엇을 해야 할지를 작성)
				>>> 복잡도 ↓, 이해도 ↑, 가독성 ↑

		> 자바스크립트를 이용한 컴포넌트 기반 아키텍쳐
				>>> 코드 재사용이 쉬워서 코드량 감소

		> 강력한 추상화
				>>> 강력한 문서 모델 추상화
					: 내부의 인터페이스는 숨기고, 정규화 과정을 거친 합성 메서드와 속성을 제공

	## 1-3-2 속도와 테스트 용이성
		> React의 가상 DOM은 자바스크립트 메모리에만 존재, 데이터를 변경하면 React는 가상 DOM을 먼저 비교하고, 렌더링 변경이 필요한 경우에만 실제 DOM에 렌더링
			→ 꼭 필요한 부분만 갱신하여 내부 상태(가상 DOM)와 뷰(실제 DOM)를 같게 만들어 성능이 우수함
		> 헤드리스 브라우저 없이도 단위 테스트 수행 가능

	## 1-3-3 React의 폭넓은 개발 커뮤니티와 생태계
```

```
# 1-4 React의 단점
	> 모든 기능을 갖춘 프레임워크는 아님, 다른 라이브러리를 함께 사용해야 할 수도 있음
	> 성숙하지 않음
	> React에 정통한 수준이 될 만한 모범 사례나 자료 등이 부족
	> 단방향 데이터 바인딩만 제공
	> 리액티브 프로그래밍이라고 볼 수는 없음
	  >> 리액티브 프로그래밍이나 아키텍처는 더 이벤트 기반이고, 탄력성과 반응성이 뛰어남
```

```
# 1-5 웹 애플리케이션에 React 적용하기
	> React는 UI 라이브러리일 뿐이므로 어떤 형태의 백엔드나 프론트엔드 데이터 관리 라이브러리와도 함쎄 사용할 수 있음
		- UI 라이브러리로 React와 관련된 Redux나 React Router를 활용한 SPA
		- MVC의 V를 대체하는 UI 라이브러리로 기존 MVC 프레임워크와의 결합
		- jQuery를 기반으로 서버 측 렌더링을 거친 애플리케이션에서 자동완성 등 일부 기능을 위한 UI 컴포넌트로 활용
		- 대부분의 로직을 직접 처리하는 전통적인 방식의 백엔드에서 서버 측 렌더링 템플릿 라이브러리로 활용
		- 백엔드와 프론트엔드에서 모두 자바스크립트를 사용하는 경우(ex; express-react-views를 활용한 Express.js. 백엔드)
		- React Native를 UI 라이브러리로 사용한 모바일 앱
		- 여러 가지 렌더링 대상에 적용할 목적으로 사용하는 UI 라이브러리

	# 1-5-1 React 라이브러리와 렌더링 대상
		> 버전 0.14 이후부터 기존 React를 react와 react-dom이라는 두 패키지로 분리해서 npm에 배포

	# 1-5-2 SPA과 React
		> SPA(단일 페이지 애플리케이션) 아키텍처는 서버보다는 클라이언트, 즉 브라우저 측에 로직이 더 많은 팻 클라이언트
		> SPA는 HTML 렌더링, 입력값 검증, UI 변경 등의 기능을 브라우저에서 해결
		> 일반적인 SPA 아키텍처
		  1. 사용자가 새로운 페이지를 열기 위해 브라우저에 URL을 입력
			2. 브라우저가 URL 요청을 서버로 전송
			3. 서버는 응답으로 HTML, CSS, 자바스크립트 파일 같은 정적 자원을 보냄. 대부분 HTML은 최소한의 기본 구조만 담고 있음. 이 단계에서는 보통 "로딩 중..." 메시지나 애니메이션 GIF 등을 이용해서 페이지가 준비중이라는 것을 나타냄
			4. 자바스크립트는 SPA를 위해 필요한 자원. 자바스크립트 로드 후 추가로 AJAX나 XHR 요청을 보내 서버에서 데이터를 불러옴
			5. 데이터는 JSON, XML 등의 포맷으로 전달받음
			6. SPA에 데이터가 전달되면 사용자 인터페이스를 구성하는 HTML을 렌더링 함. SPA는 템플릿에 전달받은 데이터를 밀어넣고 브라우저 상에서 UI를 렌더링 함
			7. 브라우저 렌더링이 끝나면 SPA는 "로딩 중..." 메시지를 없애고 사용 가능한 상태가 됨
			8. 사용자가 웹 페이지를 확인

	# 1-5-3 React 개발 스택
		> 개발자들은 해당 기능을 사용할 수 있는 라이브러리와 React를 결합해서 사용함
		> 개발 스택에는 JSX도 있음
```
